Modelo de Banco de Dados para Sistema de Cinema

Visão Geral

Este documento descreve o modelo de dados para um sistema de gerenciamento de cinema, incluindo filmes, sessões, vendas de ingressos e produtos da bomboniere.

Entidades e Atributos

Filme

Atributo Tipo Descrição
ID Inteiro Identificador único do filme (PK)
Título Texto Nome do filme
Diretor Texto Nome do diretor do filme
Duração Inteiro Duração em minutos
Classificação Indicativa Texto Faixa etária recomendada
Sinopse Texto longo Breve descrição do filme
Gênero Texto Categoria do filme
Data de Estreia Data Data de lançamento
Status de Exibição Booleano Indica se está em cartaz

Sala

Atributo Tipo Descrição
ID Inteiro Identificador único da sala (PK)
Número Inteiro Número identificador da sala
Capacidade Inteiro Quantidade de assentos
Tipo Texto Ex: Regular, VIP, 3D, IMAX
Status Operacional Booleano Indica se a sala está operacional

Assento

Atributo Tipo Descrição
ID Inteiro Identificador único do assento (PK)
Fileira Caractere Letra da fileira
Número Inteiro Número do assento na fileira
Tipo Texto Ex: Regular, VIP, Acessível
ID Sala Inteiro Referência à sala (FK)

Sessão

Atributo Tipo Descrição
ID Inteiro Identificador único da sessão (PK)
Data Data Data da sessão
Horário de Início Hora Hora de início da sessão
Horário de Término Hora Hora de término da sessão
Valor Inteira Decimal Preço do ingresso inteiro
Valor Meia Decimal Preço da meia-entrada
Idioma Texto Idioma da exibição
Formato de Exibição Texto Ex: 2D, 3D, IMAX
ID Filme Inteiro Referência ao filme (FK)
ID Sala Inteiro Referência à sala (FK)

Cliente

Atributo Tipo Descrição
ID Inteiro Identificador único do cliente (PK)
Nome Texto Nome completo
CPF Texto Documento de identificação
Email Texto Endereço de email
Telefone Texto Número de telefone
Data de Nascimento Data Data de nascimento
Data de Cadastro Data Data de registro no sistema

Funcionário

Atributo Tipo Descrição
ID Inteiro Identificador único do funcionário (PK)
Nome Texto Nome completo
CPF Texto Documento de identificação
Cargo Texto Função no cinema
Data de Admissão Data Data de contratação
Salário Decimal Remuneração
ID Supervisor Inteiro Referência ao funcionário supervisor (FK)

Ingresso

Atributo Tipo Descrição
ID Inteiro Identificador único do ingresso (PK)
Tipo de Ingresso Texto Ex: Inteira, Meia
Data e Hora da Compra Data/Hora Momento da compra
Forma de Pagamento Texto Ex: Dinheiro, Cartão, PIX
Valor Decimal Preço pago
Código de Validação Texto Código para validar o ingresso
ID Sessão Inteiro Referência à sessão (FK)
ID Cliente Inteiro Referência ao cliente (FK)
ID Assento Inteiro Referência ao assento (FK)

Produto

Atributo Tipo Descrição
ID Inteiro Identificador único do produto (PK)
Nome Texto Nome do produto
Descrição Texto longo Detalhes do produto
Categoria Texto Ex: Alimento, Bebida, Combo
Valor Decimal Preço de venda
Quantidade em Estoque Inteiro Quantidade disponível

Venda

Atributo Tipo Descrição
ID Inteiro Identificador único da venda (PK)
Data e Hora Data/Hora Momento da venda
Valor Total Decimal Valor total da compra
Forma de Pagamento Texto Ex: Dinheiro, Cartão, PIX
ID Funcionário Inteiro Referência ao funcionário responsável (FK)
ID Cliente Inteiro Referência ao cliente (FK, pode ser nulo)

Item de Venda

Atributo Tipo Descrição
ID Inteiro Identificador único do item (PK)
Quantidade Inteiro Número de itens
Valor Unitário Decimal Preço por unidade
Subtotal Decimal Valor total do item (quantidade x valor unitário)
ID Venda Inteiro Referência à venda (FK)
ID Produto Inteiro Referência ao produto (FK)
## Relacionamentos entre Entidades

Filme → Sessão: Um Filme pode ter várias Sessões, mas cada Sessão está associada a apenas um Filme.

Sala → Sessão: Uma Sala pode abrigar várias Sessões em horários diferentes.

Sala → Assento: Uma Sala contém vários Assentos, cada Assento pertence a uma única Sala.

Assento → Ingresso: Um Assento pode estar associado a vários Ingressos (em diferentes sessões).

Sessão → Ingresso: Uma Sessão pode ter vários Ingressos vendidos.

Cliente → Ingresso: Um Cliente pode comprar vários Ingressos para diferentes sessões.

Cliente → Venda: Um Cliente pode fazer várias Compras na bomboniere, mas uma compra pode existir sem identificação do cliente.

Funcionário → Funcionário: Um Funcionário pode ser supervisor de outros Funcionários.

Funcionário → Venda: Um Funcionário pode realizar várias Vendas.

Venda → Item de Venda: Uma Venda contém um ou mais Itens de produto.

Produto → Item de Venda: Um Produto pode estar presente em vários Itens de venda.

Sessão + Assento → Ingresso: Cada Ingresso está associado a uma Sessão específica e a um Assento específico.
